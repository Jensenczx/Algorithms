##STACK SORT

###THINKING

由构建优先队列对于最小或最大子集区间的处理，想到通过维护一个最大堆，或者是最小堆，然后通过对堆进行下沉操作来将一个数组进行排列，优先队列的使用场景可以是当我们得到一个序列中几个最大的元素的额时候，而且这些元素不是一次全部给我们的，而是随着流进行的，那么我们就可以额通过构造一个优先队列，然后将通过不断更新优先队列来实现我们的操作，当我们要取最大数的时候，删除之后，将最后一个元素置于顶部，然后下沉得到我们想要的序列。

应用于排序之后，首先我们要建堆，通过对于中央结点下沉的方式，构建一个大堆，然后将这些元素再进取最大结点，然后交换最小结点下沉的方式不断重复，最终得到我们想要的序列。

时间复杂度：建堆(2N)，进行排序的时候，我们需要2NlgN.（nlgn）
建堆的过程，我们通过类比堆的形式，构建一个127大小的堆，我们需要32个大小为3的堆，从最小的一个堆的构建成本开始，一个需要多少次比较，多少次交换来衡量。对于2NlgN,N前面还是有参数的，有一些元素的下降可能不到N层。
